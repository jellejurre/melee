# Melee Scaffold Skill

This skill sources the project's virtual environment and runs `tools/scaffold.py` on the specified file(s) to add function stubs based on the decomp report.

## Setup

One-time setup (if not already done):

```bash
cd /home/unity/Projects/melee/meleeDecomp
python3 -m venv .venv
source .venv/bin/activate
pip install -r reqs/base.txt
```

## Usage

Provide the file path(s) you want to scaffold. The skill will:
1. Source the `.venv` virtual environment
2. If a `.c` file is provided, automatically also scaffold the matching `.h` file
3. Run `tools/scaffold.py` with all file paths

### Example File Paths

- `src/main.c` → also scaffolds `src/main.h`
- `src/melee/lab/lab.c` → also scaffolds `src/melee/lab/lab.h`

### How to Use

When you want to scaffold a file, tell me which file(s) you want to run scaffold.py on. For example:

- "Scaffold src/melee/lab/lab.c" (will also do lab.h)
- "Run scaffold on src/main.c and src/main.h"
- "Add function stubs to src/melee/ft/ftlib.c"

The skill will execute:
```bash
source .venv/bin/activate && python .pi/skills/melee-scaffold/scaffold_wrapper.py <file_path>
```

## Notes

- When scaffolding a `.c` file, the skill automatically finds and scaffolds the matching `.h` file (same name, `.h` extension)
- Files must be relative to the project root or absolute paths
- The script appends function declarations to the end of the file
- For `.c` files: adds `/// #{func_name}` comments
- For `.h` files: adds `/* XXXXXX */ UNK_RET func_name(UNK_PARAMS);` declarations
- Requires `build/GALE01/report.json` to exist (generated by the build system)
